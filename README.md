设计原则：

- 找出应用中可能需要变化之处，把他们独立出来，不要和那些不需要变化的代码混在一起
- 针对接口变成，而不是针对实现编程
- 多用组合，少用继承
- 为了交互对象之间的松耦合设计而努力
- 封装变化
- 多用组合，少用继承
- 针对接口编程
